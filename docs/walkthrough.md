## OSS 文件读取与加载性能优化完成

为了提升页面加载速度并减少 Loading 遮罩对用户体验的干扰，已完成以下优化：

### 1. 持久化缓存 (Session Storage)
- **秒开体验**：引入了基于 `sessionStorage` 的客户端缓存。当用户再次进入页面或刷新时，系统会立即显示上次获取的数据，数据展现几乎零延迟。

### 2. 非阻塞后台加载 (SWR 模式)
- **静默更新**：采用 Stale-While-Revalidate 策略。页面显示缓存数据的同时，会在后台异步从 OSS 获取最新数据。
- **智能 Loading**：优化了全屏加载逻辑。只有在本地完全没有数据（首次访问）时才会显示 Loading 动画；后续更新均在后台完成，不干扰用户操作。

### 3. 接口聚合优化 (Composite API)
- **减少请求次数**：在 `batchFetch` 后端接口中新增了 `fullBracket` 聚合模式。原本需要由前端发起 2 次串行请求的操作，现在改为由后端一次性聚合读取所有对阵和阶段数据。
- **性能提升**：减少了客户端与服务器之间的网络往返（RTT），显著降低了网络波动对加载速度的影响。

### 4. 优化范围
- 淘汰赛对阵图页面 (`brackets/page.tsx`)
- 参赛人员/裁判列表页面 (`participants/page.tsx`)
- 全局轮询 Hook (`use-polling.ts`)

## 验证建议
- **首次访问**：清除浏览器 Session 存储后访问，确认显示一次全屏 Loading。
- **再次访问**：点击侧边栏或刷新页面，确认对阵图/列表数据瞬间出现，不再显示全屏 Loading。
- **数据同步**：在后台文件更新后，停留一段时间或手动点击刷新，确认数据能正常在后台静默更新。
